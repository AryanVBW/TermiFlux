Usage Guide

Learn how to use TermiFlux to switch between ARM64 and x86_64 architectures

Basic Usage

After installing TermiFlux, you can use it in the following ways:

Method 1: Run TermiFlux Directly

Simply run TermiFlux from your terminal:

TermiFlux

This will present you with a menu to select your desired architecture:

Menu Options:

  1. ARM64 (Default) - Switch to the native ARM64 architecture
  2. x86_64 (via Rosetta 2) - Switch to x86_64 architecture using Rosetta 2
  3. Exit - Exit the TermiFlux menu

Method 2: Using Aliases

TermiFlux installs convenient aliases for quick switching:

arm # Switch to ARM64 mode
x86 # Switch to x86_64 mode

Advanced Usage

Running Specific Applications in Different Architectures

You can run specific applications in a different architecture without switching your entire terminal:

arch -x86_64 /path/to/application # Run a specific application in x86_64 mode

Checking Current Architecture

To check which architecture you're currently using:

arch

Expected Output:

  • arm64 - You're running in ARM64 mode
  • i386 - You're running in x86_64 mode

Troubleshooting

Common Issue: Command Not Found

If you get a "command not found" error when trying to run TermiFlux:

  1. Make sure you've restarted your terminal after installation
  2. Try sourcing your shell configuration file:
    source ~/.zshrc
  3. Check if the TermiFlux script exists in /usr/local/bin:
    ls -la /usr/local/bin/TermiFlux

Common Issue: Permission Denied

If you get a "permission denied" error:

  1. Check the permissions of the TermiFlux script:
    ls -la /usr/local/bin/TermiFlux
  2. Fix permissions if needed:
    sudo chmod 755 /usr/local/bin/TermiFlux
