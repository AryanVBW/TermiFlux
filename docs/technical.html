Script Components

1. TermiFlux.sh (Core Script)

The main script that provides the architecture switching functionality.

Rosetta 2 Management

check_rosetta() { echo "Checking if Rosetta 2 is installed..." if /usr/bin/pgrep oahd >/dev/null 2>&1; then echo " Rosetta 2 is already installed." else echo " Rosetta 2 is not installed. Installing now..." softwareupdate --install-rosetta --agree-to-license if [ $? -eq 0 ]; then echo " Rosetta 2 installation successful." else echo " Rosetta 2 installation failed. Please install manually." exit 1 fi fi }

Architecture Detection

check_arch() { arch_output=$(arch) echo "Current architecture: $arch_output" }

Terminal Management

open_x86_terminal() { echo "Opening a new x86_64 terminal..." osascript -e 'tell application "Terminal" to do script "arch -x86_64 /bin/zsh"' }

2. install_TermiFlux.sh (Installer Script)

Handles the installation and setup of TermiFlux on your system.

Key Functions

  • Checks for and installs Homebrew if needed
  • Installs required dependencies
  • Downloads the TermiFlux script to /usr/local/bin
  • Sets appropriate permissions
  • Configures shell startup for auto-launch

Architecture Overview

How TermiFlux Works

TermiFlux leverages macOS's built-in architecture switching capabilities to provide a seamless experience:

  1. Rosetta 2 Integration: Checks for and installs Rosetta 2 if needed for x86_64 emulation
  2. Shell Environment Management: Configures the shell environment for architecture switching
  3. Command Execution: Uses the arch command to execute processes in the desired architecture

Architecture Switching Commands

# Switch to ARM64 exec /bin/zsh # Switch to x86_64 arch -x86_64 /bin/zsh

System Requirements

Hardware Requirements

  • Apple Silicon Mac (M1, M2, or newer)

Software Requirements

  • macOS 11 (Big Sur) or newer
  • Terminal access
  • Internet connection (for installation)
